The query

—

There is no SQL engine and no database. This is what the chart is asking for — written the way you'd ask a database, to show that the same question gets answered by file layout instead of a query planner.

How it was answered

Every zoom re-runs this. The footer's per-row-group min/max stats say which row groups can be skipped; Parquet's columnar layout means unread columns are never fetched; both turn into HTTP Range requests.

✷ = resampled in-browser from a stored tier — no Parquet tier of its own. 2 min / 5 min roll up from the stored 1min tier and 4 hours from 1h, using the same aggregation that builds the tiers. It only works while the source read stays bounded — pick a ✷ resolution zoomed out and it says so: that's where a precomputed tier earns its place.
